유효성을 처리할 수 있는 각각의 Validator들을 관리하고, 각각의 Validator의 유효성을 검증하는 유틸성 모듈

Description

jindo.m.Validator를 상속받은 각각의 Validator를 관리하고, 유효성을 검증하는 static 유틸 모듈이다.
Validation의 구조는 다음과 같다.


기능설명

  • Validator 유효성 검증 기능
  • Validator 관리
  • Validate 문법해석
  • 기본 Validator 제공
  • 사용자 Validator 제공

유효성 검증하기

유효성을 검증하기 위해서는 유효성 문법과, 유효성을 검증할 문자가 필요하다.

유효성 문법

  • 유효성의 종류는 URL, Email과 같은 Validator 종류이다.
  • 형식은 유효성의 종류에 따라 필요할 수도 있고, 필요하지 않을 수도 있다.
  • 형식이 있을 경우, 입력된 값에 형식을 적용하여, 유효성 여부를 점검한다.
  • "유효성종류:형식" 은 한개 이상이 존재 할 수 있으며, 각각의 구분은 ";"으로 구분한다.

유효성 검증하기

jindo.m.Validation.validate으로 유효성을 검증한다.

사용자 Validator

사용자 Validator를 만들어 jindo.m.Validation에서 관리할 수 있다.

사용자 Validator 만들기

  • 생성하는 Validator는 jindo.m.XxxxValidator 형태로 작성한다.
  • jindo.m.Validator를 상속받는다
  • validate을 구현한다
    정제된 문자열은 sCorrectedValue로 반환한다. 정제된 문자가 없을 경우는 null을 반환한다.
    정제된 문자열 기준으로 Validate한다. Validate결과는 bValid로 반환한다
  • validate은 htValidate객체({bValid : 성공여부, sCorrectedValue : 수정된 값})를 반환한다.

사용자 Validator 사용하기

  • jindo.m.Validation.add를 호출하여 생성한 사용자 Validator를 추가한다
  • jindo.m.Validation.validate로 추가한 Validator를 사용한다
