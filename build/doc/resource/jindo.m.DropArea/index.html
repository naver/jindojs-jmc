jindo.m.DropArea

Version : 1.4.0
DropArea 컴포넌트는 DragArea 컴포넌트로 드래그된 엘리먼트가 드랍되었을 때 지정한 동작을 수행할 수 있도록 도와주는 컴포넌트

Sample

Description

DropArea컴포넌트는 DragArea 컴포넌트로 드래그된 엘리먼트가 드랍되었을 때 지정한 동작을 수행할 수 있도록 도와주는 컴포넌트다.
DropArea 컴포넌트는 단독으로 사용할 수 없고, DragArea 컴포넌트와 함께 Drag&Drop을 기능을 완성한다.

DragArea 인스턴스에서 발생하는 모든 커스텀 이벤트를 DropArea 컴포넌트에서도 발생시켜 따로 DragArea 인스턴스를 핸들링 하지 않고 DropArea 컴포넌트에서 커버가능하다.

[기준레이어안의 엘리먼트 구조]

위의 그림과 같이 기준 엘리먼트에는 드롭 영역이 여러개 존재 할 수있다.

드롭
드래그 영역이 드롭 가능한 영역이다.
[prefix]+'area' 클래스명을 갖는다.
드롭영역이면서 드래그영역도 가능하다.

커스텀 이벤트 발생 순서

아래 그림처럼 커스텀 이벤트 발생한다.


[커스텀이벤트 발생순서]

handleDown, dragStart, beforeDrag, drag, dragEnd, handleUp의 경우 DragArea에서 발생한 커스텀 이벤트를 DropArea에서 발생시킨다.
드래그앤드롭 기능을 활용할때 따로 DragArea의 커스텀 이벤트를 이용하지 않고 DropArea의 커스텀 이벤트에서 핸들링 가능하다.

Notice

√ 지원 브라우저

  • 안드로이드 2.1 이상
  • iOS 3.0 이상

√ 주의사항

bUseTouchPoint 옵션값

drag엘리먼트가 drop 엘리먼트의 위에 있는지를 (over되어 있는지) 판단하는 기준이다
true 일 경우 사용자 터치의 좌표를 기준으로 over를 판단한다.
false일 경우 drag엘리먼트가 drop 엘리먼트와 겹쳐져 있는지를 가지고 판단한다

위의 그림과 같은 상황이라면,
bUseTouchPoint값이 true 일 경우 drop 엘리먼트에서 "over" 커스텀 이벤트는 발생하지 않는다.
bUseTouchPoint값이 false 일 경우 drop 엘리먼트에서 "over" 커스텀 이벤트는 발생한다.

Initialization

HTML
  • 기준 엘리먼트를 기준으로 컴포넌트 내부에서 드롭 엘리먼트를 찾는다
  • 드롭 대상 엘리먼트의 경우 [prefix]+'area' 클래스명을 갖는다.
Javascript
oDragInstance 옵션에 jindo.m.DragArea 인스턴스를 설정해야 한다.
