jindo.m.LayerEffect

Version : 1.4.0
특정 Layer에 애니메이션 효과를 적용하여 보여주거나, 숨기거나, 이동시키는 컴포넌트

Sample

Description

CSS3 사용
CSS3를 사용하여 애니메이션을 구현하여 모바일 환경에서 빠르게 동작한다.

브라우저 지원한계
각 애니메이션 별로 지원하는 브라우저가 다를 수 있으므로 적용시에 확인이 필요하다.

Notice

√ 지원 브라우저

  • 안드로이드 2.1 이상
  • iOS 3.0 이상

√ 주의사항

iOS 3.0
"pop-in" / "pop-out" 적용시 scale의 허용범위는 "0.1 ~ 1"이다.
iOS 3.0의 경우 scale "0"을 허용하지 않는다.
- iOS 3.0 : 0.1 ~ 1
- 그 외 : 0 ~ 1

√ 마크업 주의사항

expand, contract 메소드의 경우 레이어의 높이값, 넓이값을 조정한다.
레이어의 padding, border 값이 설정될 경우 높이값,넓이값을 설정하는데 문제가 될 수 있다.(그럴경우 전체를 감싸는 div를 다시 설정)

√ 레이어 사이즈 관련 주의사항

LayerEffect의 함수 사이즈를 조정하는 메소드들은 각 레이어의 사이즈를 계산이후에 설정한다.
퍼센트 단위가 아닌 px 단위로 계산하여 조정하는 작업을 하기 때문에
화면 리사이즈에 따른 레이어 사이즈가 바뀔 경우 "setSize()" 함수를 호출하고 Effect 함수를 실행해야 한다.

Custom Event

Name Desc Custom Event Object Properties
beforeEffect 애니메이션 효과가 시작하기 직전 발생한다.
  • sType (String) : 커스텀이벤트명(before)
  • elLayer (HTMLElement) : 애니메이션 효과가 적용된 레이어 엘리먼트
  • sEffect (String) : 적용할 애니메이션 효과 이름 , '-'을 구분한다. (fade-in, slide-left)
  • nDuration (Number) : 애니메이션 적용 시간(ms)
  • stop (Function) : 수행시 애니메이션 효과 시작되지 않는다.
  커스텀이벤트 핸들링 예제 (fade-in 효과시 애니메이션 시작하지 않음)
afterEffect 애니메이션 효과가 종료된 직후 발생한다.
  • sType (String) : 커스텀이벤트명(before)
  • elLayer (HTMLElement) : 애니메이션 효과가 적용된 레이어 엘리먼트
  • sEffect (String) : 적용할 애니메이션 효과 이름 , '-'을 구분한다. (fade-in, slide-left)
  • nDuration (Number) : 애니메이션 적용 시간(ms)
  • stop (Function) : stop를 호출하여 영향 받는 것은 없다.
  커스텀이벤트 핸들링 예제 (slide 애니메이션 이후에 fade-out 애니메이션을 호출)
