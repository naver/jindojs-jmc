페이지의 고정영역 내부를 터치하여 스크롤링 할 수 있고, 인덱스 표기 기능 및 스크롤바가 있는 컴포넌트

Sample

Description

jindo.m.IndexScroll 컴포넌트는 jindo.m.Scroll을 상속받아 구현한 컴포넌트로서,
인덱스 표기 기능을 사용할 수 있다.


[컨텐츠 구조]


기능 설명
  • 터치로 세로 영역의 스크롤이 가능하다.
  • 스크롤시 가속여부(bMomentum)에 따라 가속이 가능하다.
  • 인덱스를 표기할 수 있다.
  • 스크롤바를 표시할 수 있다.

Notice

√ 지원 브라우저

  • 안드로이드 2.2 이상
  • iOS 3.0 이상

√ 주의사항

jindo.m.IndexScroll은 jindo.m.Scroll을 상속 받아 동일한 메소드와 옵션, 기능을 제공한다.
단, 아래 사항에 대해서는 제약이 있다.

  • 가로스크롤을 사용할 수 없다. 오직 세로스크롤만 가능하다.
    (bUseHScroll 옵션이 동작하지 않는다)
  • Css3d 모드를 사용할 수 없다. 모두, bUseCss3d : false인 모드만으로 동작한다.
    (bUseCss3d 옵션이 동작하지 않는다)
  • Pull Down/up 기능을 사용할 수 없다.
    (bUsePullDown, bUsePullUp, fnPullDownIdle, fnPullDownBeforeUpdate, fnPullDownUpdating, fnPullUpIdle, fnPullUpBeforeUpdate, fnPullUpUpdating 옵션이 동작하지 않는다)

Initialization

HTML
IndexScroll 컴포넌트는 Scroll 컴포넌트와 동일하게 최상위 Wrapper과 그 자식 Scroller div로 구성되며, 실제 스크롤링 하고자 하는 대상은 Scroller div하위에 존재한다.

  • Scroller는 Wrapper의 첫번째 엘리먼트가 Scroller가 된다.
  • 인덱싱 하고자하는 엘리먼트를 Scroller 내부에 추가 할 수 있다.
  • 이때, 추가되는 인덱스 엘리먼트의 class는 [prefix] + index이다.

최상위 Wrapper는 내부적으로 overflow:hidden 속성이 설정된다.
만약, Wrapper의 position이 static일 경우는 positon:relative으로 설정된다.
Scroller는 position:absolute 속성이 설정된다.
Scroller 하위는 position:static, position:relative 속성만 사용할 수 있다.

Javascript
최상위 Wrapper를 생성자의 첫번째 파라미터로 받는다.

Option

jindo.m.Scroll의 옵션과 동일하다

Name Desc Default
bUseIndexView 인덱스뷰를 보여준다.

인덱스뷰에 표기될 내용은 기본적으로 [sClassPrefix]index로 지정된 엘리먼트의 text 정보로 구성되어 진다.
인덱스의 text정보와 다르게 다르게 인덱스뷰를 구성하기 위해서는 [sClassPrefix]index로 지정된 엘리먼트에 data-text 속성을 지정하여 표시할 인덱스이름 정보를 변경할 수 있다.

인덱스뷰의 디자인은 [sClassPrefix]indexview, [sClassPrefix]indexview-item 클래스 통해 조절가능하다.
