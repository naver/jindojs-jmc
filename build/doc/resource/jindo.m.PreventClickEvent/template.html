안드로이드 이벤트 투과 방지 컴포넌트

Sample

Description

안드로이드에서는 z-Index 상 맨 앞에 있는 레이어에서 발생한 이벤트가 투과되어 뒤에 존재하는 레이어에 이벤트가 발생하는 문제가 있다. 이를 방지하기 위한 컴포넌트이다.

기능

  • 안드로이드 투과 이벤트 문제를 방지한다
  • 기준엘리먼트 안에 있는 특정엘리먼트에 대해서는 투과가 가능하도록 한다

Notice

√ 지원 브라우저

  • 안드로이드 2.1 이상
  • iOS 3.0 이상

√ 주의사항

안드로이드
z-Index 상 맨 앞에 있는 레이어에서 발생한 이벤트가 투과되어 뒤에 존재하는 레이어에 이벤트가 발생하는 문제가 있다.
넥서스S(Android 2.3), 갤럭시S(Android 2.2), 갤럭시Tab(Android 2.2)
pass가 가능한 엘리먼트 하위에 있는 엘리먼트가 선택되는 이슈가 있다

Initialization

Javascript
생성자의 첫 파라미터로 지정된 el 엘리먼트에서는 모든 이벤트(스크롤, 터치, 클릭)가 발생하지 않는다.
el 엘리먼트 내부의 특정 엘리먼트에서 이벤트 발생을 위해서는 [sClassPrefix] + "except" 클래스를 지정한다.

Option

Name Desc Default
sClassPrefix 이벤트 통과 엘리먼트 지정 Class Prefix evt-
bActivateOnload 컴포넌트 로드시 activate 여부 true

Custom Event

Name Desc Custom Event Object Properties
pass 기준엘리먼트에서 이벤트가 통과된 경우, 사용자 이벤트가 발생
  • sType (String) : 커스텀이벤트명
  • wel ($Element) : 이벤트가 통과된 엘리먼트

  • sType (String) : 커스텀이벤트명
  • wel ($Element) : 이벤트가 중지된 엘리먼트
